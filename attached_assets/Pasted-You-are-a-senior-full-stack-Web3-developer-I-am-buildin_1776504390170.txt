- Single port entry point: Express serves API + Vite frontend via proxy

## Core Backend Modules to Build

### 1. DID Generation (server/src/lib/did.ts)
- Generate W3C DID Document: did:polygon:<walletAddress>
- Ed25519 keypair generation using @noble/ed25519
- Store DID document in PostgreSQL + anchor hash on Polygon
- DID resolution endpoint: GET /api/did/:did

### 2. VC Issuance Engine (server/src/modules/credentials/)
- Issue JWT-VC signed with Ed25519
- Store encrypted credential on IPFS (Web3.storage)
- Anchor credential hash on Polygon via CredentialAnchor contract
- Return VC to holder wallet
- Endpoints:
  POST /api/credentials/issue
  GET  /api/credentials/:id
  POST /api/credentials/verify
  POST /api/credentials/revoke/:id
  GET  /api/credentials/holder/:did

### 3. Smart Contracts (contracts/)
\`\`\`solidity
// CredentialAnchor.sol
// anchorCredential(bytes32 credHash, string did, string vcId)
// isAnchored(bytes32 credHash) returns (bool)
// getAnchor(bytes32 credHash) returns (Anchor memory)

// RevocationRegistry.sol  
// revokeCredential(bytes32 credHash, string reason)
// isRevoked(bytes32 credHash) returns (bool)

// TrustRegistry.sol
// addIssuer(address issuer, string did, string orgName)
// isTrustedIssuer(address issuer) returns (bool)
// getIssuerScore(address issuer) returns (uint256)
\`\`\`

### 4. Blockchain Service (server/src/lib/polygon.ts)
- ethers.js v6 Polygon Mumbai/Mainnet client
- Contract interaction wrappers for all 3 contracts
- Transaction receipt polling with retry logic
- Gas estimation before sending

### 5. Auth Module (server/src/modules/auth/)
- Two auth methods:
  a) MetaMask signature verification (Web3 wallet auth)
  b) Phone OTP fallback (for non-crypto users)
- JWT session after auth
- RBAC middleware checking User.role

### 6. WebSocket Server (server/src/lib/websocket.ts)
- ws library (compatible with Replit)
- Rooms per userId for targeted notifications
- Events: credential_issued, verification_request, 
  payment_due, scheme_matched
