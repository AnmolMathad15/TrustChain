## PHASE 1: Decentralized Identity (DID)

### Requirements:

* Generate a DID for each user during registration
* DID format: did:example:<unique-id> (or did:ethr for Ethereum-based)
* Store DID in database linked to user
* Create DID Document structure (JSON) including:

  * id (DID)
  * publicKey
  * authentication
* Add API endpoints:

  * POST /api/did/create
  * GET /api/did/:userId

### Frontend:

* Replace “SSI ID” with DID in profile page
* Display DID QR code for sharing identity

---

## PHASE 2: Verifiable Credentials (VC)

### Requirements:

* Convert all user documents (Aadhaar, PAN, certificates, health records) into Verifiable Credentials

* VC structure must include:

  * @context
  * type
  * issuer
  * issuanceDate
  * credentialSubject (user DID + data)
  * proof (signature)

* Hash each credential using SHA-256

* Store:

  * Full VC in DB/IPFS
  * Hash on blockchain

### Backend APIs:

* POST /api/credentials/issue
* GET /api/credentials/:userId
* POST /api/credentials/verify

### Frontend:

* Update Documents module:

  * Show “Verified Credential” badge
  * Add “Share via QR” button
  * Add “Verify” button
